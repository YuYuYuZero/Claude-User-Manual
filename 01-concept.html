第一章 · 什么是 Claude

很多新用户的第一个困惑是：“Claude 到底是一个东西还是一堆东西？” 打开文档，满屏的 API、Code、Skills、MCP、Harness，像是被一堆陌生名词迎面拍过来。

本章不逐一讲解这些名词，而是先用一个类比在 3 分钟内把它们挂到一个熟悉的场景上，再回来填细节。

1.1 一个类比：Claude 是“电力公司 + 电器门店”

想象你家里要用电。你有两个选择：

  • 你可以拉一条电线直接从电网取电，用来驱动你自己 DIY 的各种设备。这就是 Claude API(Messages API)——直接调用模型，你自己搭应用。
  • 你也可以买一台电饭煲，插上电就能煮饭。这就是 Claude Code——一个开箱即用的、专门用来“写代码”的成品工具。
  • 电饭煲里有预设菜单（“煲粥”/“蒸蛋”/“糙米饭”），按一下就执行。这就是 Agent Skills——把某类任务封装成可复用的流程。
  • 电饭煲底下的温控芯片、过热保护、防干烧电路。这就是 Agent Harness——你平时看不见，但没有它整个东西会烧穿。

类比的映射质量：4 关系同时对应

Claude 产品用电场景
Claude API（模型调用接口）直接从电网拉电线，用它驱动自定义设备
Claude Code（终端编程工具）买成品电饭煲，插上电就用
Agent Skills（可复用技能）电饭煲里的预设菜单
Agent Harness（安全调度层）电饭煲内部的温控和防干烧电路
类比的边界（诚实标注）：电是“单向流动”的能源，而 Claude 是双向对话的模型——你给它内容，它返回内容。这个类比在“消费者可选多种形态”上精确，在“信息流向”上偏简化。

1.2 两种核心产品形态：API vs Managed Agents

当你决定“用 Claude”时，第一个要做的决策是选哪种使用方式：

它是什么直接调用模型的接口，你自己写应用代码去驱动它预构建、可配置的 Agent Harness，跑在托管基础设施上
适合什么场景自定义 Agent 循环、细粒度控制、短任务、实时对话长时间运行的任务、异步工作

1.3 五个必须建立的心智模型

阅读本手册剩余章节前，请先把下面五个词装进脑子。后面所有操作都围绕这五个词展开。

1. Model（模型）
Claude Opus / Sonnet / Haiku 是三档不同能力的模型。Opus 最强最贵，Haiku 最快最便宜。一次调用要指定用哪个（见附录 A）。
2. Context Window（上下文窗口）
模型的“工作记忆”。Claude Opus 4.7 可达 1M tokens，相当于 ~555k 英文单词或 ~2.5M unicode 字符。超过就装不下。
3. Tool Use（工具使用）
Claude 不只能说话，还能“调工具”——网络搜索、代码执行、文件读写等。调用时返回 tool_use 结构化请求。
4. Skill（技能）
可复用的“领域说明书 + 脚本 + 资源”，通过渐进式披露（progressive disclosure）分三级加载，不挤占上下文。
5. Harness（脚手架）
裹在模型外的“操作系统”：管理记忆、安全拦截、工具调度，让长任务不失控。

1.4 为什么选 Claude（说服性内容）

1.4.1 适用与不适用场景一览

适用场景：Claude Opus 4.7 的 1M tokens 上下文窗口支持一次性载入整本书（约 555k 英文单词），可靠知识截止 2026 年 1 月，适合长文档分析、多轮工具调用、复杂代理编码。

不适用场景：一次性短问答（用 Haiku 4.5 更便宜，$1/$5 per MTok）；需要离线运行（Claude 目前不支持本地部署）。

1.4.2 三个典型场景与对应模型

典型场景推荐模型理由
分析一份 300 页合同，找出所有风险条款Claude Opus 4.71M 上下文装得下整份合同；128k 最大输出能写出详细分析报告
日常编程：代码重构、跑测试、写提交信息Claude Sonnet 4.6速度快、价格适中（$3/$15 per MTok）；支持手动扩展思考
批量处理 1 万条客服工单的分类标签Claude Haiku 4.5最快、最便宜（$1/$5 per MTok）；批量场景下总成本下降约 80%

1.4.3 Claude 做不了什么（诚实边界）

  • 不能实时访问互联网：除非显式启用 web_search 工具，否则 Claude 依赖训练数据（Opus 4.7 截止到 2026 年 1 月）
  • 不能执行任意代码：除非显式启用 code_execution 工具，且沙盒有限制
  • 不保证输出 100% 正确：事实性任务（法律、医疗、财务）必须人工复核
  • 部分新模型不支持手动扩展思考或预填充：详见第四章和第六章
