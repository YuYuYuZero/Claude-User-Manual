附录 B · 术语表

本手册全程对同一个概念只用一个名字。下面是完整的“用词表”，也是你 review 团队成员的写作时可以对照的检查清单。

B.1 模型与调用

中文术语英文 / 代号定义
模型Model一个具体的 AI 模型版本，如 Claude Opus 4.7。调用时通过 model 字段指定
模型 IDModel ID模型的完整标识，如 claude-opus-4-7。有时带日期后缀（claude-haiku-4-5-20251001）
模型别名Model Alias不带日期的简称，如 claude-haiku-4-5。推荐在代码里用别名而非日期 ID
Messages APIMessages API直接调用模型的 HTTP 接口，传 messages 数组，拿 content 回复。无状态
SDKSDKPython / TypeScript / Java 等官方客户端库，封装了 Messages API 的 HTTP 细节
CLICLI命令行工具 ant，用 shell 直接调 API，不需要写代码

B.2 上下文与记忆

中文术语英文定义
上下文窗口Context Window模型一次能“看见”的 token 总量，相当于工作记忆。Opus 4.7 为 1M tokens
上下文衰退Context Rot上下文变长后准确率和召回率下降的现象——“越长 ≠ 越好”
上下文感知Context Awareness较新模型能“看见”自己剩余 token 预算，合理分配算力
压缩Compaction服务器端对旧对话自动摘要、释放上下文。Beta 特性
上下文编辑Context Editing通过策略自动清除工具结果、思考块等冗余内容
TokenToken模型处理文本的最小单位，大致 1 个汉字 ≈ 1 token，1 个英文单词 ≈ 0.75 token

B.3 提示与对话

中文术语英文定义
系统提示System Prompt发送前就注入的指令，定义角色、规则、输出格式。通过 system 字段传递
用户消息User Messagerole: "user" 的消息，用户本轮输入
助手消息Assistant Messagerole: "assistant" 的消息，模型的历史回复
预填充Prefill在 messages 最后一位放 assistant 消息作为前缀，强制输出格式。新模型部分不支持
停止原因Stop Reason响应里的 stop_reason 字段，常见值：end_turn（正常结束）、tool_use（要调工具）、max_tokens（被截断）
提示缓存Prompt Caching把稳定前缀缓存下来，降低重复调用的输入 token 成本。支持 5 分钟或 1 小时

B.4 推理与思考

中文术语英文定义
扩展思考Extended Thinking模型在给最终答案前，先输出 thinking 块展示推理过程
自适应思考Adaptive Thinking模型自己决定何时思考、思考多深。Opus 4.7+ 的推荐模式
思考预算Budget Tokens扩展思考最多花多少 token。budget_tokens 必须 < max_tokens
思考块Thinking Block响应内容里 type: "thinking" 的块。工具使用场景下必须原样回传
交错思考Interleaved Thinking允许模型在工具调用之间再思考。Claude 4 模型支持

B.5 工具与技能

中文术语英文定义
工具使用Tool Use模型通过 tool_use 块调用外部函数或能力
客户端工具Client-side Tool你的代码执行的工具。模型返回 tool_use，你跑完后传回 tool_result
服务器端工具Server-side Tool平台侧执行的工具，你直接拿结果。如 web_search、code_execution
工具结果Tool Result工具执行完后的输出，通过 tool_result 块传回
Skill / 技能Agent Skill包含 SKILL.md + 可选脚本/资源的模块化能力包。有 name 和 description
渐进式披露Progressive DisclosureSkills 三级加载机制：元数据 → 指令 → 资源，按需读入不挤占上下文
MCPModel Context Protocol连接 Claude 与外部工具/数据源的开放协议
MCP 连接器MCP Connector直接从 Messages API 连到远程 MCP 服务器的能力

B.6 架构与工程

中文术语英文定义
Harness / 脚手架Agent Harness裹在模型外的“操作系统”：记忆管理 + 安全 + 调度 + 监控
人机回环Human-in-the-loop高风险操作强制暂停、等人类确认的机制
Sub-AgentSub-Agent把长任务拆成多个专职子 Agent（调研 / 写代码 / 测试），互相传精简 JSON
记忆压缩Memory CompactionHarness 阶段性清空历史、只留结构化摘要，防止上下文爆炸
内存工具Memory ToolClaude 跨对话存储和检索信息的工具，构建长期知识库

B.7 数据与合规

中文术语英文定义
零数据保留ZDR / Zero Data RetentionAPI 响应返回后数据不落地。部分功能不支持 ZDR，使用前需要确认
数据驻留Data Residency用地理控制决定推理在哪个区域跑。通过 inference_geo 参数
Beta 标头Beta Header启用预览功能所需的 HTTP 标头，如 skills-2025-10-02

B.8 配置与环境

中文术语英文 / 文件定义
API 密钥API Key调用服务所需的凭证。通过 x-api-key 头或 ANTHROPIC_API_KEY 环境变量传递
项目级配置.claude/settings.json项目根目录下的配置，优先级高于全局
全局配置~/.claude/settings.json用户主目录下的配置，所有项目默认生效
项目规则CLAUDE.mdClaude Code 每次启动自动读取的项目规则文件
全局规则~/.claude/CLAUDE.md用户级默认规则，所有项目适用
写作用词约定：本手册全程遵守以下规则——
  • 第二人称统一用“你”，不混用“用户”、“您”、“开发者”
  • 所有命令、文件名、API 字段用 等宽代码 格式
  • 任务标题以动词开头（如“安装 Node.js”，不是“Node.js 安装”）
  • 多义词一经确定不再改（如“Skill” 一律译作“技能”或直接保留英文，不穿插其他译法）
