第八章 · 可用性测试报告

8.1 测试目标与方法

手册的最终质检是真实用户能否仅凭它完成任务。为此，我们邀请 2 位未参与本课程的同学，让他们仅凭本手册执行下面 3 个具体任务，全程录屏 + 录音，事后做简短访谈。

8.1.1 测试任务

编号任务描述对应章节预期完成时间
T1在 macOS 上跑通第一次 Claude API 调用，看到返回 JSON第二章 · 快速入门5–10 分钟
T2装好 Claude Code CLI，在一个 git 项目里启动它并完成第一次对话第三章 · 3.2 / 3.415–20 分钟
T3在 Claude Code 里创建一个 CLAUDE.md，然后让 Claude 按规则执行一次任务第三章 · 3.910–15 分钟

8.1.2 测试者画像

👩 测试者 A · 王同学，新闻学专业研三
  • 技术背景：几乎零基础。会用网页版 ChatGPT，没装过命令行工具，听说过 API 但没调过
  • 设备：MacBook Air M2，macOS 14.5
  • 测试任务：T1 + T2（跳过 T3，因不熟悉编辑器）
👨 测试者 B · 李同学，数据科学专业研一
  • 技术背景：用过 Python 和 pip，会基本 git，没用过 npm 和 Node.js
  • 设备：MacBook Pro M3，macOS 15.1
  • 测试任务：T1 + T2 + T3（全部）

8.1.3 测试方法

  1. 静默观察：测试者操作时，观察员不提示、不解答，只记录卡顿点（回看哪一段、停顿多久、面部表情）
  2. think-aloud：鼓励测试者一边操作一边说出“我现在在想什么、为什么这么做”
  3. 事后访谈（约 10 分钟）：问 5 个开放性问题——而非“你觉得怎么样”:
    • 哪一步让你最想放弃？
    • 有没有哪个名词你看了三遍还不懂？
    • 有没有按手册做了但结果跟手册不一致的地方？
    • 如果让朋友用这本手册，你会先警告他什么？
    • 你觉得最有用的一节是？最没用的一节是？

8.2 测试发现：5 个问题清单

下面按严重程度排序，标签 高 表示“足以让测试者放弃任务”，中 表示“需要回看 2 次以上”，低 表示“小困惑、不影响完成”。

高问题 P1：测试者 A 在第二章配 API 密钥时卡住 8 分钟

现象：她不知道 export ANTHROPIC_API_KEY=... 这条命令应该“在哪里输入”。原文只写了“环境变量”，但她不知道终端是什么。

触发段落：§2.1 Tab 1 步骤 1

访谈原话：“它说设置环境变量，但我电脑上没看到‘环境变量’这四个字的设置入口。”

高问题 P2：测试者 B 装 Claude Code 后报 EACCES 权限错

现象：他用了系统自带 Node（brew 装的），跑 npm install -g @anthropic-ai/claude-code 报权限错，他凭直觉加 sudo 装上了。但后续遇到全局包冲突。

触发段落：§3.2.2

访谈原话：“手册说推荐 nvm，但我已经有 brew 装的 node 了，不知道要不要切。”

中问题 P3：两位测试者都问过“~/.claude/settings.json 是什么意思”

现象：~/（用户主目录）对完全零基础用户不直观。测试者 A 在 Finder 里找 ~ 文件夹找不到。

触发段落：§3.3

中问题 P4：测试者 B 完成 T3 后问“CLAUDE.md 改了之后要不要重启 Claude Code?”

现象：手册没明确说“修改 CLAUDE.md 后下次启动才生效”。他改完后期待 Claude 立刻反应。

触发段落：§3.9

低问题 P5：测试者 A 完成 T1 后说“看不懂返回的 JSON 含义”

现象：她跑通了请求，看到 JSON 返回，但不知道哪个字段是“答案”、哪个字段是“账单”。

触发段落：§2.1 Tab 1 步骤 3

8.3 改进措施（已落地到第二、三章）

问题改进方案已落地章节
P1 不知道在哪输入命令第二章开头加一行“打开 macOS 自带的【终端】App（Cmd+空格 搜 Terminal），所有命令都在这里输入”§2.1 之前
P2 用 sudo 硬装第三章 §3.2.2 加 warning 框：“如果是已有 brew Node 又遇到 EACCES，先 brew uninstall node 再装 nvm，不要 sudo 硬装”§3.2.2 + 第六章 t1
P3 不懂 ~/ 是什么第三章 §3.3 第一次出现 ~/.claude/settings.json 时加注：“~ 表示你的用户主目录，例如 /Users/yourname/”§3.3 注释
P4 改 CLAUDE.md 后不知要重启§3.9 末尾加 tips 框：“修改 CLAUDE.md 后，下次 claude 启动时才会重新加载；当前会话不受影响”§3.9 末尾
P5 看不懂返回 JSON§2.1 步骤 3 末尾加一句：“content[0].text 是答案；usage 是 token 账单（决定费用）”§2.1 Tab 1 步骤 3

8.4 修改前后对比（展示 2 处）

对比一：§2.1 步骤 1 加入“在哪输入命令”

修改前
设置 API 密钥（环境变量） 
export ANTHROPIC_API_KEY='your-api-key-here'

持久化：把这一行追加到 ~/.zshrc……

修改后
设置 API 密钥（环境变量）

📌 所有命令都在 macOS 自带的【终端】里运行：Cmd+空格，输入 Terminal 回车打开它。

export ANTHROPIC_API_KEY='your-api-key-here'

持久化：把这一行追加到 ~/.zshrc（~ 是你的用户主目录，例如 /Users/yourname）……

对比二：§3.9 末尾加“何时生效”提示

修改前

优先级：项目级 CLAUDE.md > 用户全局 ~/.claude/CLAUDE.md。

该文件建议提交到 Git，让团队成员共享同一套规则。

修改后

优先级：项目级 CLAUDE.md > 用户全局 ~/.claude/CLAUDE.md。

该文件建议提交到 Git，让团队成员共享同一套规则。

💡 何时生效：修改 CLAUDE.md 后，下次 claude 启动时才会重新加载；当前会话不受影响。

8.5.1 关键结论

  1. 第二章是“留存断点”——零基础用户最容易在这里放弃。改进 P1+P5 之后预计完成率 +30%
  2. 第六章常见问题被高度认可——测试者 B 在装包时遇到 EACCES，直接翻到第六章 t1 自救成功
  3. “何时生效”类信息普遍缺失——除了 CLAUDE.md，后续可补充“修改 settings.json 后是否要重启”“加白名单后立即生效吗”等

8.5.2 下一轮测试计划

  • 招募 1 位完全没装过 npm/node 的 Windows 用户，验证手册的跨平台描述是否够清晰（目前手册偏向 macOS）
  • 请测试者 B 在改进版手册上重做 T2，看 EACCES 解释是否避免了 sudo 误操作
  • 把 P5（JSON 字段含义）的改进搬到第二章其他语言 Tab(Python / TypeScript / Java)，保持一致
