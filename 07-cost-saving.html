第七章 · 省钱与成本控制

Claude Opus 4.7 的定价是输入 $5/MTok，输出 $25/MTok。一个不小心，一下午就能烧掉几十刀。本章 8 个具体技巧能帮你省下 80% 的费用。

7.1 三大原则：别走弯路

原则一：不是所有活都需要 Opus

日常 90% 的任务，Sonnet 4.6 够用了——它的价格是 Opus 的 1/5。

/model sonnet            # 切到 Sonnet
/model opus              # 遇到难题再临时切回 Opus,完事再切回去

在 ~/.claude/settings.json 里设置默认模型：

{
  "model": "claude-sonnet-4-6"
}

原则二：effort 不是越高越好

xhigh 和 max 贵得离谱——它们让模型“反复想反复验证”，每个动作都消耗大量 token。

任务复杂度推荐 effort
简单改动（改一行字、加注释）low
日常开发medium
不行再升high

原则三：上下文不是越大越好

每条消息都会带完整的上下文重新计算。100k tokens 的上下文，每问一个问题就是 $0.5 的输入成本。

所以 /clear 和 /compact 要勤快用——这不只是“防止变傻”，更是直接省钱。

7.2 8 个具体省钱技巧

1

用 @文件 引用，别让 Claude 搜索

需要 Claude 看某个具体文件时，直接用 @路径 指明，比让它自己找省得多：

❌ 烧 token 的写法
> 看看 auth 模块怎么实现的

→ Claude 会跑几次 Glob / Grep 找文件，可能读 5–8 个无关文件才定位到目标，token 哗哗烧。

✅ 省 token 的写法
> 看看 @src/auth/login.ts 怎么实现的

→ 直接给路径，Claude 一步到位读到目标文件，零搜索成本。

2

CLAUDE.md 保持简洁（200 行以内）

每条消息都加载它。200 行 CLAUDE.md，按每行 20 tokens 算，一次对话就是 4000 tokens 的固定开销。控制好长度，把详细文档放到 @docs/ 里按需引用。

3

关掉不用的 MCP 服务器

每个连上的 MCP 服务器都会在系统 prompt 里注入说明文字、占用 token。用不到的就断开：

/mcp remove <id>
4

用 subagent 处理重活

长任务里需要大量读代码、找 bug 的部分，用 subagent 处理：

/agents

subagent 在独立上下文里跑，结果只带“精华”回来，不污染主会话——既省 token 又防止主会话变长后变傻。

5

长会话及时压缩

看到上下文 > 80%，立刻 /compact。不要等系统提示 "context full" 才动手——那时已经多烧了几刀。

带 focus 参数的 compact 比裸 compact 好用得多：

/compact 保留所有关于认证错误的细节
/compact retain the database schema and migration history
6

用 --max-budget-usd 设上限

让 Claude 自动跑长任务时，启动时带上预算：

claude --max-budget-usd 5.00 "重构这个模块"

用满 $5 自动停。再也不会出现“睡前让它跑、醒来发现烧了 $80”的事故。

7

把重复工作封装成 /命令

一件事你做了 3 次，就把它变成 /命令（见第四章 §4.8）。

每次用命令的 prompt 是固定的，不会因为你每次表达不同而浪费 token。命令还可以指定 model: claude-sonnet-4-6 强制用便宜模型。

8

/cost 常看，长会话每隔一会儿瞄一眼

/cost

会显示当前会话的 token 用量和费用：

Session cost:
  Input tokens:  145,230
  Output tokens: 18,542
  Total cost:    $0.79

Plan usage today: $4.23 / $50.00

知道自己花了多少，才有底——这是养成“成本意识”的第一步。

7.3 实际花费参考

下表是真实使用的经验值，帮你预估自己一个月可能花多少钱：

场景模型 + effort每小时大约成本
日常问答、代码补全Sonnet + low$0.50 – $2
一般开发、修 bugSonnet + medium$2 – $8
复杂重构、大功能Opus + high$10 – $30
长程 agent（连续跑几小时）Opus + xhigh$30 – $100
套餐参考：一个 Max $100/月 的计划大概对应每月 $500–$800 的 API 额度用量，够大部分重度开发者用。新手建议先用 Pro 套餐或 API 按量付费几周，知道自己用量水平再决定要不要上 Max。

7.4 一份“省钱清单”：每天开工前检查

养成习惯，每天开第一个 Claude Code 会话之前问自己：
  1. 今天的活复杂吗？如果是日常开发，切到 sonnet
  2. 会跑多久？如果可能跑 1 小时以上，带 --max-budget-usd 上限
  3. 有重复工作吗？如果是第 3 次了，封装成 /命令
  4. 项目的 CLAUDE.md 还在 200 行以内吗？定期 review，删冗余
  5. 会用 @文件路径，还是会让 Claude 自己 grep？能 @ 就 @

7.5 读完本章，你应该能做

  • ✅ 解释为什么“上下文越大越花钱”，并能用 /compact 主动控制
  • ✅ 知道自己日常 90% 的活该用什么模型 + effort 组合
  • ✅ 启动 Claude Code 时会用 --max-budget-usd 设上限
  • ✅ 至少封装过一个项目级 /命令 用来减少重复输入
  • ✅ 会用 /cost 实时监控，知道自己每小时大概花多少
