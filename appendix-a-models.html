A.2 选型决策树

  1. 不知道选哪个 → 从 Claude Opus 4.7 开始。跑通效果后，不满意再换
  2. 要手动控制思考预算(budget_tokens) → Claude Sonnet 4.6 或更早的 Claude 4 模型
  3. 成本敏感的大批量短任务（如标签分类、简单摘要） → Claude Haiku 4.5
  4. 长文档分析 / 一次吞整份合同 → Opus 4.7 或 Sonnet 4.6（都有 1M 窗口）

A.6 单请求大小限制

  • Opus 4.7 / 4.6 / Sonnet 4.6：单请求最多 600 张图像或 PDF 页面
  • 200k 上下文窗口的模型（如 Haiku 4.5）：单请求最多 100 张图像或 PDF 页面
  • 发送大量图像或大型文档时，可能先撞到“请求大小限制”，而不是 token 限制

A.7 批处理 API 的额外能力

在批处理 API 上，Opus 4.7、Opus 4.6 和 Sonnet 4.6 通过指定一个 beta 标头 output-300k-2026-03-24，可支持单次请求最多 300k 输出 tokens（超过同步 API 的 128k / 64k 限制）。

批处理 API 调用的成本比标准 API 低 50%，适合对延迟不敏感、但量大的场景（如批量分类、离线分析）。
